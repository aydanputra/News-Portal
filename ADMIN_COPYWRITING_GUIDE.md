# Panduan Copywriting Admin

Dokumen ini menjadi acuan istilah UI agar konsisten di seluruh halaman admin, terutama modul yang sering dipakai tim editorial.

## Tujuan

- Menjaga konsistensi label antar halaman Create, Edit, dan List.
- Menghindari istilah campur (misalnya Tipe/Jenis, Meta Info/Meta Data).
- Mempercepat review UI karena bahasa antarmuka sudah baku.

## Istilah Baku Utama

| Konteks | Istilah Baku | Hindari |
| --- | --- | --- |
| Variasi entitas | Tipe | Jenis |
| Metadata konten | Meta Data | Meta Info |
| URL tujuan | Link URL (Opsional) | Link Tujuan |
| Script iklan | Script (Google AdSense) | Script (Google Adsense), Kode Script (untuk label tipe) |
| Input script | Kode Script (HTML/JS) | Tempel Kode Script |
| Status penayangan | Status | Status Aktif |
| Tipe iklan gambar | Gambar / Banner | Banner saja |
| Pustaka aset | Pustaka Media | Media Library |
| Elemen judul kedua | Subjudul | Subtitle |

## Gaya Penulisan

- Gunakan Title Case ringan untuk label pendek: `Tipe Iklan`, `Kode Script`, `Jadwal Tayang`.
- Untuk bantuan konteks, gunakan tanda kurung: `(Opsional)`, `(HTML/JS)`, `(Google AdSense)`.
- Gunakan kata kerja aktif pada helper text: `Pastikan kode script aman dan valid.`
- Gunakan istilah Indonesia untuk navigasi utama, pertahankan istilah teknis umum: `Script`, `URL`, `HTML/JS`.

## Pola Khusus Modul Ads

- List: `Nama Iklan`, `Tipe`, `Posisi`, `Jadwal`, `Status`, `Aksi`.
- Create/Edit:
  - `Tipe Iklan`
  - `Gambar / Banner`
  - `Script (Google AdSense)`
  - `Link URL (Opsional)`
  - `Kode Script (HTML/JS)`
  - `Status`

## Checklist Saat Menambah UI Baru

- Apakah label memakai istilah baku pada tabel di atas?
- Apakah halaman Create dan Edit menggunakan istilah yang sama?
- Apakah penulisan AdSense konsisten (`AdSense`, bukan `Adsense`)?
- Apakah helper text ringkas dan tidak ambigu?
